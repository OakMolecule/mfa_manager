# VaultX UI 设计规范

**版本**: v1.0  
**日期**: 2026-04-20  
**参考原型**: `docs/ui_mockup.html`

---

## 1. 设计语言

VaultX 采用 **Material Design 3 Blue** 风格：

- 蓝色系主色调，传递信任与安全感
- 圆角卡片布局，现代简洁
- 高对比度文字，无障碍友好
- 支持亮色 / 暗色双模式

---

## 3. 字体

```
系统字体栈（无需引入外部字体）:
'Segoe UI', 'PingFang SC', 'Microsoft YaHei', sans-serif

字号规范:
- 页面标题 (topbar):     18px / 600
- 卡片主标题:            15px / 600
- TOTP 验证码:          22px / 800 / letter-spacing: 5px
- 正文:                 14px / 400
- 次要信息:             13px / 400
- 标签 / Badge:         11-12px / 500-600
- 密码字体:             monospace / 13px / letter-spacing: 1px
```

## 4. 圆角规范

| 组件 | 圆角值 |
|------|--------|
| 卡片 (card) | 12px |
| 按钮 (button) | 8px |
| 输入框 (input) | 8px |
| 对话框 / 抽屉 | 16px |
| 头像 / 图标 | 50%（圆形）|
| 复制图标按钮 | 5-6px |

## 5. 间距系统

基础单位 4px，常用值：

| 用途 | 值 |
|------|-----|
| 卡片内边距 | 12-16px |
| 表单组间距 | 16px |
| 列表项间距 | 8-10px |
| 行内元素间距 | 6-8px |
| 卡片阴影 | `0 1px 4px rgba(0,0,0,0.1)` |
| 卡片阴影（悬停）| `0 3px 10px rgba(0,0,0,0.1)` |

### 6.7 Badge（计数徽章）

## 7. 页面规范

### Screen 1：解锁页

**布局**: 全屏渐变背景 + 居中卡片

```
背景: linear-gradient(135deg, #E3F2FD 0%, #FAFAFA 60%, #E8F5E9 100%)
卡片宽度: 380px
卡片圆角: 16px
卡片阴影: 0 8px 40px rgba(25,118,210,0.18)
```

**组件**:
- Logo 图标（68×68px，圆角 20px，蓝色渐变）
- 主密码输入框（带 👁 显示/隐藏）
- 错误提示条（红色背景，显示剩余尝试次数）
- 解锁按钮（全宽 Primary Button）
- 创建新金库按钮（全宽 Outlined Button）

**交互**:
- 输入框回车触发解锁
- 错误 5 次后按钮禁用，显示退避倒计时

---

### Screen 2：主列表页

**布局**: 顶栏 + 侧边栏（200px）+ 内容区（滚动）

**顶栏组件**:
- 🔐 Logo 图标
- 搜索框（圆角 18px，白色半透明背景）
- ＋ 新建按钮
- ⚙ 设置按钮
- 🔒 锁定按钮

**侧边栏**:
```
宽度: 200px
背景: var(--sidebar-bg)
节: 全部条目 / 类型（密码/TOTP）/ 分类
激活项: 背景 primary-container + 左侧 3px 主色条
```

**条目卡片结构（内联展示）**:
```
┌──────────────────────────────────────────────────────┐
│ [头像] 标题  🔑🔐    domain.com    时间戳    [›]     │  ← 可点击跳详情
├──────────────────────────────────────────────────────┤
│ 🔐  482 917  ████░░░ 18s                        [📋] │  ← TOTP置顶（仅当有TOTP时）
├─────────────────────────┬────────────────────────────┤
│ 👤 user@example.com [📋]│ 🔑 ••••••••••  [👁] [📋]  │  ← 用户名+密码同行
└─────────────────────────┴────────────────────────────┘
```

- TOTP 行：22px / 800 字体，进度条宽 44px，倒计时
- 用户名行：13px，flex:1 溢出省略
- 密码行：monospace，默认遮盖，👁 切换
- 无 TOTP 时只显示底部一行（用户名+密码）

**FAB 按钮**: 右下角 ＋，56×56px，主色圆形

---

### Screen 3：条目详情页

**布局**: 顶栏（带返回、编辑、删除）+ 侧边栏 + 内容区

**内容区结构**:
- 详情头部卡片（头像 + 标题 + 分类 Chip）
- 字段卡片：网址 / 用户名 / 密码（带强度条）/ TOTP 区块（如有）/ 备注
- 每个字段右侧有复制按钮

---

### Screen 4：TOTP 总览页

**布局**: 顶栏 + 3 列网格

**TOTP 卡片**:
```
背景: var(--card-bg)
圆角: 12px
即将过期（≤5s）: 边框 #FF6D00, 背景 #FFF3E0
验证码字体: 26px / 800 / letter-spacing: 4px
进度条: 5px 高，主色（即将过期时橙色）
```

---

### Screen 5：密码生成器（抽屉）

**布局**: 覆盖在当前页面右侧，背景模糊 + 半透明遮罩

```
抽屉宽度: 300px
圆角: 16px（左侧两角）
```

**控件**:
- 长度滑块（8-64，实时更新预览）
- 5 个切换开关（大写/小写/数字/特殊字符/排除歧义字符）
- 密码预览区（monospace，深色背景）
- 刷新 / 复制 / 使用此密码按钮

---

### Screen 6：设置页

**布局**: 顶栏（带返回）+ 侧边栏 + 内容区

**设置分组卡片**:
- 🔒 安全（修改密码 / 自动锁定 / 剪贴板清除 / 错误次数）
- 🎨 外观（主题：亮色 / 暗色 / 跟随系统）
- 💾 数据管理（导出加密备份 / 导入 / 导出明文-危险操作）
- ℹ 关于（版本号 / 加密算法）

---

### Screen 7：新建条目页

**布局**: 顶栏（✕关闭 + 保存按钮）+ 侧边栏 + 内容区

**表单结构**:
1. 标题输入（必填）
2. 🔑 密码区块卡片（默认展开，右上角开关可关闭）
   - 网址、用户名/邮箱、密码（带⚡生成按钮）、强度条
3. 🔐 TOTP 双因素验证区块卡片（默认折叠，开关展开）
   - 密钥（Secret）、发行方（Issuer）、账号
4. 分类 Chip 选择器
5. 备注文本域

> **密码和 TOTP 可同时开启**，开关互相独立。

## 8. 动效规范

遵循"高级动画效果可无，但圆角配色需有"原则：

| 效果 | 时长 | 缓动 |
|------|------|------|
| 颜色过渡（主题切换）| 300ms | ease |
| 按钮悬停背景 | 150-200ms | ease |
| 卡片悬停上移 | 150ms | ease（translateY -1px）|
| 侧边栏激活背景 | 150ms | ease |
| 开关滑块移动 | 200ms | ease |
| 复制反馈（变绿）| 即时 → 2000ms 后复原 | — |
