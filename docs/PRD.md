**版本**: v1.0

**状态**: 已评审

## 1. 产品概述

### 1.1 产品定位

VaultX 是一款**本地优先**的跨平台密码管理工具，支持 Linux / macOS / Windows。所有数据加密存储在用户本机，不依赖云服务，不收集任何遥测数据。

### 1.2 核心价值

- **安全**：AES-256-GCM 加密 + Argon2id 密钥派生，军事级保护
- **便捷**：列表页直接显示用户名、密码、TOTP 验证码，一键复制
- **统一**：密码与 TOTP 可在同一条目中并存，无需在两个工具间切换
- **单文件**：所有数据存储在一个 `.vaultx` 加密文件中，便于备份

### 1.3 目标用户

- 需要管理多个账号的开发者与运维人员
- 注重安全且不信任云服务的隐私用户
- 需要在工作机和个人机之间手动同步密码库的用户

## 2. 功能需求

### 2.1 核心功能

#### F1 - 密码管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F1.1 | 创建、读取、编辑、删除密码条目 | P0 |
| F1.2 | 字段：标题、网址、用户名/邮箱、密码、备注 | P0 |
| F1.3 | 密码强度实时检测（弱/中/强）| P0 |
| F1.4 | 内置密码生成器（长度 8-64，可配置字符集）| P0 |
| F1.5 | 一键复制用户名、密码（剪贴板自动清除）| P0 |

#### F2 - TOTP 管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F2.1 | 在任意条目中附加 TOTP 密钥（与密码并存）| P0 |
| F2.2 | 实时计算并显示 6 位 TOTP 验证码 | P0 |
| F2.3 | 显示剩余有效时间与进度条 | P0 |
| F2.4 | 即将过期（≤5s）时高亮橙色警告 | P1 |
| F2.5 | 一键复制验证码 | P0 |
| F2.6 | TOTP 总览页：网格展示所有绑定 TOTP 的条目 | P1 |

#### F3 - 主界面

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F3.1 | 列表页直接展示：TOTP 验证码（置顶大字）、用户名、密码（默认遮盖）| P0 |
| F3.2 | 点击 👁 图标切换密码明文/遮盖 | P0 |
| F3.3 | 点击 📋 图标复制对应字段 | P0 |
| F3.4 | 关键词搜索（标题、用户名、网址）| P0 |
| F3.5 | 侧边栏按类型（密码/TOTP）和分类（工作/个人/金融/购物）过滤 | P1 |

#### F4 - 安全

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F4.1 | 主密码 Argon2id 密钥派生（m=65536 KiB，t=3，p=4）| P0 |
| F4.2 | AES-256-GCM 加密存储，nonce 每次保存随机生成 | P0 |
| F4.3 | 内存中敏感数据使用 `zeroize` 归零（解锁→锁定时）| P0 |
| F4.4 | 连续密码错误 5 次后指数退避锁定（2^n 秒，最大 64s）| P0 |
| F4.5 | 可配置自动锁定（默认 5 分钟无操作）| P1 |
| F4.6 | 复制后剪贴板自动清除（默认 30 秒）| P1 |

#### F5 - 数据管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F5.1 | 导出加密备份（`.vaultx` 格式）| P0 |
| F5.2 | 导入 `.vaultx` 文件恢复数据 | P0 |
| F5.3 | 危险操作：导出明文 JSON（需二次确认）| P1 |

#### F6 - 设置

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F6.1 | 修改主密码 | P0 |
| F6.2 | 主题切换（亮色 / 暗色 / 跟随系统）| P1 |
| F6.3 | 自动锁定时间配置 | P1 |
| F6.4 | 剪贴板清除延迟配置 | P1 |

## 3. 数据模型

### 3.1 金库文件格式（`.vaultx`）

```json
{
  "version": 1,
  "argon2_params": {
    "m_cost": 65536,
    "t_cost": 3,
    "p_cost": 4,
    "salt": "<base64>"
  },
  "nonce": "<base64, 12 bytes>",
  "ciphertext": "<base64, AES-256-GCM encrypted JSON>"
}
```

### 3.2 解密后明文结构

```json
{
  "entries": [<Entry>, ...]
}
```

### 3.3 Entry（条目）

```json
{
  "id": "<uuid-v4>",
  "title": "GitHub",
  "category": "工作",
  "created_at": "<ISO 8601>",
  "updated_at": "<ISO 8601>",
  "password": {
    "url": "https://github.com",
    "username": "oehjio@example.com",
    "password": "tK9#mZ2@pL5!xQR",
    "notes": ""
  },
  "totp": {
    "secret": "JBSWY3DPEHPK3PXP",
    "issuer": "GitHub",
    "account": "oehjio@example.com",
    "algorithm": "SHA1",
    "digits": 6,
    "period": 30
  }
}
```

> `password` 和 `totp` 字段均为可选（`null`），但至少一个不为空。

## 4. 非功能需求

### 4.1 性能

- Argon2id 解锁耗时：目标 0.3-1.0 秒（可接受范围，阻止暴力破解）
- 列表页渲染 1000 条目：< 100ms
- TOTP 计时器刷新精度：误差 < 100ms

### 4.2 兼容性

| 平台 | 最低版本 |
|------|---------|
| Linux | glibc 2.31（Ubuntu 20.04+）|
| macOS | 10.15 Catalina |
| Windows | 10（1903+）|

### 4.3 包体积

- 目标：< 15 MB（单可执行文件，含所有资源）

### 4.4 可靠性

- 保存失败时不破坏原文件（先写临时文件，成功后原子替换）
- 启动时校验文件 HMAC，文件损坏时报错而非静默失败

## 6. 状态机

```
[首次启动] ──创建金库──▶ [解锁界面]
[再次启动]              ──▶ [解锁界面]
[解锁界面] ──正确密码──▶ [列表页]
[解锁界面] ──错误≥5次──▶ [指数退避锁定]
[列表页]   ──超时/锁定──▶ [解锁界面]（内存清零）
[列表页]   ──选择条目──▶ [条目详情]
[列表页]   ──点+按钮──▶ [新建条目]
[条目详情] ──点生成──▶ [密码生成器抽屉]
[任意页面] ──点设置──▶ [设置页]
```
