# MFA Manager 产品设计文档（桌面端）

## 1. 文档信息

- 产品名称：`MFA Manager`
- 文档类型：产品设计文档（Product Design Doc）
- 版本：`v1.0`
- 设计基线：`index.html`（单文件 HTML/CSS/JS 原型）
- 目标平台：桌面端（Desktop First）

## 2. 产品定位

`MFA Manager` 是一个本地优先（local-first）的桌面认证信息管理工具原型，用于统一管理：

- TOTP 动态验证码账户（如 GitHub、AWS、银行）
- 密码型账户（如路由器、内网等）

核心价值：

- **快**：一屏完成查看、筛选、搜索、复制
- **稳**：验证码倒计时清晰可见，信息有自动回遮机制
- **安全感**：锁屏、临时显示、剪贴板定时清空等安全交互默认启用

## 3. 目标用户与使用场景

## 3.1 目标用户

- 同时管理多个 2FA 账户的开发者、运维、金融/办公人员
- 需要在桌面环境高频复制验证码与密码的用户
- 希望通过分组与筛选快速定位账户的重度用户

## 3.2 典型场景

- 登录云平台时，快速查看并复制 6 位 TOTP 验证码
- 登录旧系统时，查看并复制保存的密码
- 在多账户环境下，通过“工作/财务/个人”分组快速切换
- 临时离开工位前，一键锁屏保护敏感信息

## 4. 产品目标与边界

## 4.1 当前阶段目标（原型目标）

- 验证桌面端信息架构与交互流是否顺畅
- 验证 TOTP 与密码两类账户在同一界面共存时的可用性
- 验证安全交互（自动回遮、剪贴板清空、锁屏）用户理解度

## 4.2 非目标（本阶段不做）

- 不实现真实 TOTP 算法与后端存储
- 不实现多设备同步、云备份、团队共享
- 不接入真实 QR 扫描器与系统级生物认证

## 5. 信息架构与页面结构

## 5.1 桌面布局

应用窗口固定为桌面形态（约 `1100×700`），三层结构：

1. **顶部标题栏**：应用名、窗口控制视觉元素、锁定与更多操作
2. **左侧边栏**：
   - 一级导航：账户 / 分组 / 安全 / 设置（当前为原型入口）
   - 分类筛选：全部、工作、财务、个人（含数量徽标）
   - 底部统计与“添加账户”主按钮
3. **右侧主内容区**：
   - 工具栏：标题、账户总数、搜索框、视图切换、排序按钮
   - 账户内容区：按分类分段显示，默认双列卡片网格

## 5.2 浮层结构

- **添加/编辑账户弹窗**：桌面居中模态（scrim + modal）
- **上下文菜单**：卡片右上角更多按钮触发
- **锁屏覆盖层**：全窗口覆盖 PIN 解锁
- **Snackbar**：底部中间轻提示

## 6. 功能设计

## 6.1 账户展示

账户分两类：

- `totp`：展示动态码、倒计时环、复制验证码
- `password`：展示密码占位、临时显示、复制密码

每张卡片包含：

- 图标、服务名称、账户标识（邮箱/用户名等）
- 类型相关信息区（动态码或密码）
- 右上角更多操作菜单（编辑、移动分组、删除）

## 6.2 分组与筛选

- 支持分类：`work`、`finance`、`personal`
- 侧边栏点击分类后，主区域仅渲染该分类分段
- “全部账户”展示所有分段
- 分类计数实时随新增/删除变化

## 6.3 搜索

- 搜索范围：服务名 + label + username
- 输入时前端实时过滤
- 当分段无可见卡片时，该分段标题与内容自动隐藏

## 6.4 TOTP 行为

- 倒计时周期：30 秒
- 圆环与数字联动更新（每秒刷新）
- 颜色状态：
  - `<=10s` 警告色
  - `<=5s` 危险色
- 点击卡片显示验证码（`secret` 格式化为 `XXX XXX`）
- 验证码显示 8 秒后自动回遮
- 周期归零时若卡片处于展开态，自动收起并回遮

## 6.5 密码行为

- 密码默认遮蔽
- 展开后可见，5 秒后自动回遮
- TOTP 卡片第三行可临时显示保存密码并支持复制

## 6.6 复制与反馈（含剪贴板策略）

- 支持复制验证码或密码到剪贴板
- 复制后按钮状态变为“已复制”
- Snackbar 提示清空时长（由策略决定）
- 到期后恢复按钮文案与图标状态

剪贴板清空策略（新增）：

- 在“设置 > 安全”提供配置项：`剪贴板自动清空`
- 配置粒度：
  - 全局开关：开启 / 关闭
  - 验证码清空时间：`10s / 15s / 25s / 30s / 60s / 永不`
  - 密码清空时间：`10s / 15s / 30s / 60s / 120s / 永不`
- 默认值（兼容当前原型行为）：
  - 验证码：25 秒
  - 密码：15 秒
- 边界规则：
  - 若设置为“永不”，复制提示改为“已复制（不自动清空）”
  - 复制同类内容时，覆盖并重置该类清空计时器
  - 应用锁屏时可选“立即清空剪贴板”（安全增强开关，默认开启）

## 6.7 添加与编辑账户

弹窗支持两种类型切换：

- TOTP
- 密码

字段与规则：

- 必填：服务名称
- 密码类型必填密码
- 可选：账户、图标、分类、密钥等
- 编辑模式复用同一表单并回填数据

交互要点：

- TOTP 模式显示扫描区和密钥字段
- 密码模式隐藏扫描区，仅保留密码相关输入
- 提交后刷新列表与分类计数

## 6.8 删除账户

- 从上下文菜单触发
- 删除后刷新列表并提示“账户已删除”
- 若删除 TOTP 账户，同步移除对应 timer 状态

## 6.9 锁屏

- 可从标题栏手动触发
- 6 位 PIN 校验（原型默认 `123456`）
- 错误显示抖动和错误提示，记录尝试次数
- 支持指纹按钮模拟快捷解锁

## 6.10 主题切换

- 支持 `a-g` 共 7 套主题
- 通过 `data-theme` 和 CSS 变量统一换肤
- 切换后计时环颜色与文字颜色同步刷新

## 6.11 KeePass 导入（新增）

目标：支持从 KeePass 迁移历史账户，降低首次使用门槛。

支持来源：

- `KeePass KDBX`（推荐，需用户提供数据库密码/密钥文件）
- `KeePass CSV`（兼容模式，提示“明文风险”）

导入入口：

- 侧边栏“设置 > 数据管理 > 导入”
- 首次启动空态页显示“从 KeePass 导入”

字段映射（默认规则）：

- `Title` -> `issuer`
- `UserName` -> `username`（同时可映射为 `label`）
- `Password` -> `password`
- `URL` -> `label`（当 `username` 为空时）
- `Notes` -> 备注（后续字段，当前可暂存）
- TOTP：
  - 优先从 `otp` / `TOTP Seed` / `otpauth` 等常见字段提取
  - 若存在 `otpauth://`，解析出 `secret/issuer/period/digits/algorithm`

导入流程：

1. 选择文件 -> 输入解密信息（如 KDBX 密码）
2. 预览映射结果（总条数、可导入条数、冲突条数、异常条数）
3. 用户确认导入策略：
   - 跳过重复
   - 覆盖已存在
   - 保留两份（自动重命名）
4. 执行导入并生成结果报告

去重与冲突规则（默认）：

- 主键建议：`issuer + username + type`
- 若 `issuer` 缺失，使用 `Title` 或 `URL` 回退
- 冲突记录写入“导入报告”，可一键回滚本次导入（事务批次）

异常处理：

- 文件损坏 / 解密失败 -> 明确错误原因与重试引导
- 字段缺失 -> 标记为“部分导入”，并提示需手动补全
- TOTP 解析失败 -> 仍可导入为密码记录，并标注“未识别 TOTP”

## 7. 交互与状态设计

## 7.1 卡片状态

- 默认态：信息遮蔽、复制按钮隐藏
- Hover 态：显示“点击查看”提示
- 展开态：显示敏感信息与复制按钮
- 已复制态：按钮视觉反馈（check + 已复制）

## 7.2 模态状态

- 关闭态：不可交互
- 打开态：遮罩 + 弹窗缩放进入
- 点击遮罩空白区关闭

## 7.3 异常与空态（建议）

当前原型已覆盖输入校验提示；建议后续补充：

- 搜索无结果空态（图标 + 文案 + 清空搜索）
- 分类无账户空态（引导添加）
- 剪贴板不可用 fallback（手动复制提示）

## 8. 视觉与设计规范（基于现实现）

- 字体：
  - 主字体：`Inter`
  - 等宽码：`Roboto Mono`
  - 图标：`Material Icons Round`
- 圆角层级：8 / 10 / 14 / 16 / 20 / 24
- 主色系统：由主题 token 驱动（primary/surface/outline 等）
- 动效：
  - 卡片浮起与按压
  - 菜单缩放
  - 模态缩放
  - 锁屏错误抖动

## 9. 数据模型（原型级）

账户模型（简化）：

- `id: number`
- `icon: string`
- `issuer: string`
- `label: string`
- `username: string | null`
- `password: string | null`
- `secret: string | null`
- `cat: 'work' | 'finance' | 'personal'`
- `type: 'totp' | 'password'`

辅助状态：

- `timers[]`：仅 TOTP 账户倒计时状态
- `currentCat`：当前分类筛选
- `isListView`：网格/列表视图切换状态
- `clipboardPolicy`：剪贴板策略（开关、验证码时长、密码时长、锁屏即清空）
- `importJobs[]`：导入任务记录（来源、状态、成功/失败数、报告路径）

## 10. 安全与隐私设计（原型）

- 默认遮蔽敏感字段
- 定时自动回遮（验证码 8s、密码 5s）
- 剪贴板定时清空（可配置，默认 25s/15s）
- 支持手动锁屏与 PIN 解锁
- 支持锁屏时立即清空剪贴板（可配置）

说明：当前为前端原型，未实现本地加密存储与密钥管理。

## 11. 可用性与性能目标（原型验收）

- 列表渲染：常规数据量下交互流畅无明显卡顿
- 倒计时刷新：1 秒粒度稳定，颜色阈值切换正确
- 核心路径（搜索、筛选、复制、编辑）可在 1-2 步完成
- 主题切换后可读性与对比度保持可用

## 12. 验收标准（UAT Checklist）

- [ ] 左侧分类切换可正确过滤右侧账户
- [ ] 搜索可实时过滤并隐藏无结果分段
- [ ] TOTP 倒计时、色阶、回遮逻辑正确
- [ ] 密码显示/回遮与复制反馈正确
- [ ] 添加、编辑、删除可正确更新列表与计数
- [ ] 锁屏可正常锁定/解锁并显示错误反馈
- [ ] 7 套主题切换无样式错乱
- [ ] 剪贴板清空策略可配置并正确生效
- [ ] KeePass 导入可完成预览、冲突处理与结果报告

## 13. 后续演进建议

1. 接入真实 TOTP 计算（RFC 6238）与 Secret 管理
2. 本地加密存储（主密码 + KDF + 安全擦除）
3. 系统托盘、小窗快速复制、全局快捷键
4. 导入/导出与备份恢复（优先支持 KeePass）
5. 与 `slint_ui` 对齐桌面 IA（侧栏、双列、模态）
